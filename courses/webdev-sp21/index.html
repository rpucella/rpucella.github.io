Textbooks
There is no required textbook. We'll mostly be using freely available resources, or documents I will share with you. If you truly want a textbook, the following is a reasonably broad overview of the field as it existed seven years ago, going into a lot of details about the basics:

  • Sebasta, Programming the World Wide Web, Pearson. 2015
The following book is the best book I know about JavaScript, and it's thin to boot. It doesn't contain the latest updates to the syntax (known as ES2020) but if you're serious about doing front-end web development, you need to read this book at least once:
  • Crockford, JavaScript: The Good Parts. O'Reilly, 2008
We're going to program our servers using Flask, a Python web framework. Plenty of documentation about Flask online. If you need a narrative, this is reasonable:
  • Grinberg, Flask Web Development. O'Reilly, 2014
Generally speaking, the technology evolves way too quickly for any good book to be up-to-date with what's currently available out there. Still, the above generally sticks to foundational topics that remains true.

  • Feb 2:HTML Overview

    Slides from the lecture.

    The MDN HTML pages that describe pretty much everything you need to know.

    What was introduced in HTML 5?

    An introduction to the Document Object Model.

    I mentioned the basic rendering algorithm implemented by browser. How browsers render various pieces of a web document is called a layout. The basic rendering algorithm with block and inline element I alluded to in class is the flow layout.

    This article describes the process of turning a web document into a rendered page in overwhelming detail. Note that it also takes styling into account.

  • Feb 4:CSS Overview

    Slides from the lecture.

    The MDN CSS pages that describe pretty much everything you need to know.

    How many CSS properties are there?

    The cascade part of CCS.

    CSS rules are defined using selectors. There is a pretty involved language of selectors that you can use in rules.

    Which CSS rule applies when multiple rules affect an element depends on the specificity of the rules involved.

    Pseudo-classes are a way to define rules to apply to certain elements in certain states.

    Distinctions between inline, inline-block, and block for the display property.
