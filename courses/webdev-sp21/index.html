Riccardo Pucella (riccardo.pucella@olin.edu)

Office hours
Wednesdays 4-5pm EST (same Zoom as class), or by appointment.

Textbooks
There is no required textbook. We'll mostly be using freely available resources, or documents I will share with you. If you truly want a textbook, the following is a reasonably broad overview of the field as it existed seven years ago, going into a lot of details about the basics:

  • Sebasta, Programming the World Wide Web, Pearson. 2015
The following book is the best book I know about JavaScript, and it's thin to boot. It doesn't contain the latest updates to the syntax (known as ES2020) but if you're serious about doing front-end web development, you need to read this book at least once:
  • Crockford, JavaScript: The Good Parts. O'Reilly, 2008
We're going to program our servers using Flask, a Python web framework. Plenty of documentation about Flask online. If you need a narrative, this is reasonable:
  • Grinberg, Flask Web Development. O'Reilly, 2014
Generally speaking, the technology evolves way too quickly for any good book to be up-to-date with what's currently available out there. Still, the above generally sticks to foundational topics that remains true.

  • Jan 28:Introduction

    Slides from the lecture.

    The history of the web from CERN.

    The first web page.

  • Feb 2:HTML Overview

    Slides from the lecture.

    The MDN HTML pages that describe pretty much everything you need to know.

    What was introduced in HTML 5? There's also a superb overview of HTML 5.

    An introduction to the Document Object Model.

    I mentioned the basic rendering algorithm implemented by browser. How browsers render various pieces of a web document is called a layout. The basic rendering algorithm with block and inline element I alluded to in class is the flow layout.

    This article describes the process of turning a web document into a rendered page in overwhelming detail. Note that it also takes styling into account.

  • Feb 4:CSS Overview

    Slides from the lecture.

    The MDN CSS pages that describe pretty much everything you need to know.

    How many CSS properties are there?

    The cascade part of CCS.

    CSS rules are defined using selectors. There is a pretty involved language of selectors that you can use in rules.

    Which CSS rule applies when multiple rules affect an element depends on the specificity of the rules involved.

    Pseudo-classes are a way to define rules to apply to certain elements in certain states.

    Distinctions between inline, inline-block, and block for the display property.

  • Feb 11:Introduction to JavaScript

    Slides from the lecture.

    The MDN JavaScript pages that describe pretty much everything you need to know.

    The current EMCAScript standard (ES2020), which includes in particular in its introduction a description of what each previous standard introduced. There is a draft of the next standard, ES2021.

    A description of the class system in JavaScript.

    A discussion of the various forms of equality in JavaScript.

    An interesting history of JavaScript.

  • Feb 16-18:The Document Object Model and Events

    Slides from the lectures.

    The demo code I played with in lecture: file dom.html; file events.html; a styled version of events.html. (If you click on these links, the browser will open them as HTML documents. To download them to your machine, right-click on the link and chose "Save Link As..." or something equivalent.)

    MDN documentation entry point on the Document Object Model.

    Key classes in the DOM: The Node class; the Element class; the HTMLElement class.

    Finding elements in the DOM: a general introduction; the getElementById() method; the querySelector() method; the querySelectorAll() method.

    A complete example of how to manipulate an HTML table in JavaScript that illustrates all the features of creating elements and adding them to the DOM tree.

    MDN introduction to events, as well as a list of all events types bucketed by category. The MDN introduction includes a good description of event propagation (where an event triggered on an element will propagate or bubble up through its parent elements).

  • Feb 23:The Model-View-Controller Design Pattern

    Slides from the lecture.

    The demo code I played with in lecture: mvc.html. (Here's the auxiliary mvc.css and mvc.js files that the document loads.)

    There isn't a concise description of MVC that I can easily point you to. Most articles either go deep into the weeds of the various flavors of MVC, or they argue that so and so that claims to be MVC is not really MVC but this other thing that if you squint you can't actually distinguish from MVC. <scorn>Pedants arguing with pedants</scorn>. The following article by Fowler, GUI Architectures, is at least tolerable, though it focuses on GUI architectures and not on MVC as we have been applying it to HTML documents and JavaScript. If you have time for it, I'd dig around Fowler's writing a bit. Much of it is interesting and useful, whether you agree with it or not.

    If you're curious, you can read the original paper describing MVC by Krasner and Pope, A Description of the MVC User Interface Paradigm in the Smalltalk-80 System. Note that it is based on Smalltalk, a delightful old object-oriented language that now lives on mostly as the source of the object-oriented component of Objective-C.

    Slides from the lectures: Introduction, Components, ReactJS in Practice, and State Management.

    Combined sample code from the introductory lecture. You may need to run a local web server to run this code. Try it directly in the browser first, and if it gives you a CORS error (or some other security failure), start a local web server and load the code through that.

    Overview of JSX.

    I did not touch functional components. Basically, if a component has no state, you can simply use a Javascript function that takes a single `props` argument and returns a JSX value. ReactJS seems to be moving in the hooks direction, though best practices haven't quite jelled yet.

  • Apr 22:Server-Side Templates

    Slides from the lectures.

    Sample code from the lecture.

    The Web Template System entry from Wikipedia contains links to server-side template engine (among other sort of templating engines, including template engines that are filled in the frontend.)

    Both JSP, ASP, and Ruby on Rails are web application server frameworks geared at providing dynamic HTML documents via templates.

    PHP is a popular template-based framework that unlike the frameworks above does not come with its own web application server infrastructure, but rather is meant to be connected to a standard web server such as Apache or nginx, which comes with modules to enable connection to the PHP template engine.

  • Apr 29:Sessions and Cookies

    Slides from the lectures.

    Sample code from the lecture.

    A short overview of statelessness and some advantages of statelessness.

    A short introduction to adding statefulness on top of the stateless HTTP protocol via cookies.

    MDN tutorial on cookies.

    RFC 6265, the precise IETF standard on state management via cookies, a surprisingly readable document once you get to Section 3.

    An entertaining and illuminating writeup by David Kristol, HTTP Cookies: Standards, Privacy, and Politics, where he discusses work on RFC 2965, the previous Cookie standard before the current RFC 6265.

  • May 4:Authentication

    Slides from the lectures.

    Sample code from the lecture.

    Wikipedia article on cryptographic hash functions that you should use instead of passwords inside a database of users, and an interesting thread on security.stackexchange on how to securely hash passwords. Turns out it's subtle. By the way, the fact that databases don't store passwords but hash is the reason why you can't ask an app to give you back your password when you've forgotten it, but rather forces you to reset your password: the app doesn't actually have your password so can't give it back to you.

    A security.stackexchange thread on session authentication versus token authentication with some interesting links.

    If you're interested in token authentication (where authentication is kept on the browser), you'll probably want to use JWT which are supported by libraries on all server-side infrastructures.

    If you're interested in using SSO (single-signon), or allowing a user to log into your web app using their Google credentials (say), then you'll want to read about OAuth.
