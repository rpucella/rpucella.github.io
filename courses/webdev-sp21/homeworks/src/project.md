# Final Project

As a final project for the course, I'm asking you to implement a web
app. That web app can be entirely a frontend, a combination of a
frontend and a backend, or a server-only API. Make it something that
you find interesting. If you need to do some web app work for a SCOPE
project, it may be possible to fold that into this project, though
we'll need to talk about it.

(I will probably post a few suggestions in the chat room and then here
in the next few days, if you're fresh out of ideas.)

I would like each team to send me an email with their proposed project
by Thursday April 1, 2021 (23h59). The email should list and CC all
team members, and include a brief description of the intended project,
including the technology you'll use.

The proposal doesn't have to be final or complete. Think of it as the
first part of a conversation. I fully expect we'll have a bit of an
exchange trying to flesh out the project and set expectations.

If you have ideas for a project, and you would like to discuss it with
me before the proposal, feel free to get in touch.

## Instructions

This project is to be done in teams of 2. (Teams of 3 are okay, but
you'll have to do more.) You may discuss problems with fellow
students, but all submitted work must be entirely your team's own, and
should not be from any other course, present, past, or future. If you
use a solution from another source please cite it.

Your project source code should contain a README text file that
lists the members of your team, your email addresses, and any remarks
that you wish to make to the instructor. The README should also contain
instructions on how to install and run your source, if applicable.

Projects include a programming component (where you create an
artifact) and a final presentation in class during the Final Events
period. Submission of the programming component should be done by
zipping up your source code and using the **Submit File**
functionality of our [chat room](https://chat.rpucella.net) by the
Final Artifact due date. Only one submission per team, please.

The final presentation should be about fifteen minutes, and
describe the problem you're trying to solve and the solution
you've implemented, including a demo. 


* * *

## Presentation Guidelines

Let's target a 15 minutes presentation.

I would ask that you prepare slides - especially since we're going to do this online.

As far as the structure of the presentation, I will not hold you to anything, but here is a common structure:

- Introduction to the web app
- Architecture (structure of the code)
- Interesting challenges of the problem or implementation
- Demo
- "What did I learn?"
