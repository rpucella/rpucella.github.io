Databases (Spring 2019)

ENGR 3599 / Thu 1:30-4pm / AC 428 304

This is an introduction to database systems. Topics covered include: the relational model, SQL-based querying, transactions, data structures, as well non-relational database models. The approach will be hands-on: rather than write code to interface to existing databases, we will write code to implement simplified versions of the various database models and their query systems.

  • Apr 11:Distributed Databases

    Distributed databases are distributed into nodes either by replication or by partitioning (or both).

    The two-phase commit protocol, which ensures that database nodes agree on whether a distributed transaction should commit or abort.

    The CAP Theorem.

    Lamport's Paxos protocol (well, protocol family) is one of the most common ways to achieve consensus on what the data stored in a distributed system is when some of the nodes disagree due to update messages not getting to them. It is worth understanding. Here is a two-parter that does a reasonable job explaining the details: Part 1, Part 2.

    Wikipedia entries on consistent hashing and distributed hash tables, the main technique for sharding in NoSQL systems.

  • Apr 25:Graph Databases

    Two excerpts from Eifrem, Webber, and Robinson's Graph Databases, which mostly discusses Neo4j, a popular graph database: the query language, and the internal representation.

    Hasura is an example of a graph-query language grafted atop a standard relational backend.

    Lecture slides describing typical graph representations and algorithms.

  • May 2:Databases and Programming Languages

    JDBC is the Java API for connecting to databases. Most databases have JDBC drivers that support that API. It has become a de facto standard, and many other languages use that same API to provide database connectivity.

    ORMs (Object-Relational Mappings) are a more abstract way to provide an interface to a database through a programming language: ORMs reify database tables as data structures in the language, so that modifying the structures will automatically update the underlying tables, and changes to the underlying tables will be visible through the structures.

    LINQ is one approach to integrate SQL-style querying within a programming language (here, C#).

    Object persistence is the idea that we can create objects in a programming language and have those objects automatically be made persistent by transparently storing them in a database. Java does not do this automatically, but serialization is the main mechanism to achieve persistence. JADE is an example of a programming language that supports object persistence natively and transparently.

    Transactional memory is an approach to programming concurrent systems in a way that does not require locks, by essentially moving the notion of transactions to the world of reading and writing memory within a single machine.
