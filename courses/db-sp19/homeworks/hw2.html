Description

In this homework, we're going to implement the relational model, and the basic operations of relational algebra.

File homework2.py contains a skeleon class for relations, the basic ingredient of the relational model. Recall that in the relational model, a (schematized) relation is a set of tuples along with a schema, in particular, the names of the attributes of the relation. For us, a relation is defined by giving the list of attributes (in the order in which they will appear in the tuples), a list of attributes defining the primary key (a subset of the attributes), and an optional list of initial tuples:

class Relation:

    def __init__ (self, columns, primary_key, tuples=[]):

        self._columns = columns
        self._primary_key = primary_key
        self._tuples = set(tuples)

    def __repr__ (self):

        result =  "------------------------------------------------------------\n"
        result += (", ".join(self._columns)) + "\n"
        result += "------------------------------------------------------------\n"
        result += "".join([ str(t)+"\n" for t in self._tuples ])
        result += "------------------------------------------------------------"
        return result

    def columns (self):

        return self._columns

    def primary_key (self):

        return self._primary_key

    def tuples (self):

        return self._tuples

All pretty straightforward -- the only moderate subtlety is that the representation of tuples is as a set. You may want to read about sets in Python as a review: tutorial section and set methods

Part B

Complete the implementation of method select(pred) which implements the selection relational algebra operation, taking a predicate and returning a new relation with the same schema as the source relation but containing only the tuples satisfying the predicate.

A predicate P is a function taking a dictionary as input and returning true or false, where a tuple satisfies predicate P if calling P with a dictionary mapping every attribute to the value in the tuple corresponding to that attribute return True. This lets us write the following to get a relation with only tuples where the value corresponding to attribute foo has value greater than 10:

r.select(lambda t : t["foo"] > 10)

or equivalently, by giving a name to the predicate

def p (t):
    return t["foo"] > 10

r.select(p)

Question 3

Add an aggregate method to the Relation class

def aggregate (self,aggr):
    pass

which takes an aggregation list aggr and returns a new relation with a single tuple in it corresponding to the aggregation of the tuples in the source relation according to aggr.

Here's what aggr looks like: it is a list of triples, each triple of the form ( name , op , attr ) where name is the name you want to give to the result of the aggregation, op is an aggregation operation (one of sum, count, avg, max, min), and attr is the name of an attribute to aggregate with operation op. For example, the triple ("total_balance","sum","balance") describes the aggregation of attribute balance with operation sum, the result of which goes into an attribute called total_balance. The aggregation list simply defines all the components of the resulting tuple, telling you how to create each component.

The attributes of the new relation are the names of the aggregated attributes. You can take the primary key attributes to be the empty list.

Some sample output:

>>> BOOKS.aggregate([("sum_pages","sum","numberPages")])
------------------------------------------------------------
sum_pages
------------------------------------------------------------
(7205,)
------------------------------------------------------------

>>> BOOKS.aggregate([("sum_pages","sum","numberPages"),("avg_pages","avg","numberPages")])
------------------------------------------------------------
sum_pages, avg_pages
------------------------------------------------------------
(7205, 450.3125)
------------------------------------------------------------

>>> BOOKS.aggregate([("sum_pages","sum","numberPages"),("count_pages","count","numberPages"),("avg_pages","avg","numberPages"),("max_pages","max","numberPages"),("min_pages","min","numberPages")])
------------------------------------------------------------
sum_pages, count_pages, avg_pages, max_pages, min_pages
------------------------------------------------------------
(7205, 16, 450.3125, 739, 181)
