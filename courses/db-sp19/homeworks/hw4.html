Description

In this homework, we're going to play a bit with storage. Just enough to give you a sense of how it works. We won't have time to get into it too too deep.

In homework4.py, I've given you some code that abstracts away from block-based files. Class File defines a file as a random-access series of blocks. The interface for a File is:

class File:

  def __init__ (self,block_size)

  def size (self)

  def allocate_block (self)
    
  def pull (self,i)
    
  def push (self,i,block)
    
  def statistics (self)

When creating a file, you specify the block size (that is, the number of tuples that fit in a block). Usually, this is fixed by the operating system, but we're abstracting away here. The size of a file is the number of blocks it contains. Method allocate_block allocates a new block to the file and adds it at the end of the file. Method pull(i) reads the block at position i (0 being the position of the first block) in memory so you can work on it. Method push(i,block) writes a block to the file at position i. In order to make a change to the file, you need to read a block, make changes to the block, and write the block back. Method statistics reports on how many block reads and writes you've done. Methods pull and push also print messages indicating what they're pulling or pushing and where. Helpful for debugging.

Class Block defines a block of tuples. The relevant interface for you is:

class Block:

  def has_space (self)
    
  def size (self)

  def last (self)
    
  def append (self,tup)
    
  def get (self,i)
    
  def put (self,i,tup)

You generally will never need to create a block (it's done by file.allocate_block()), so the above is the API to work with a block. Method has_space() checks if there's a free spot at the end of the block. Tuples are added at the end of the block with append(tup). Method size() returns the size of the block (how much space it has total) while last() returns the position of the last tuples appended. It returns -1 if no tuple was appended. Method get(i) is used to get the tuple at position i in the block (0 being the position of the first tuple). Method put(i,tup) can be used to put a tuple at a specific position in the block. Please note that this will not update the last used position in the block, so use at your own risks. You should be able to do the whole homework without put, but to reflect reality, I'm making it available.

Our goal is to create three implementations of relations that use files as storage, instead of internal tuple lists: one that uses a simple unsorted file, one that uses a file where tuples are sorted by primary key values, and one that uses a file where tuples are sorted by primary key values and an index to help find tuples by primary key value. Recall that a relation has the following basic interface:

class Relation:

    def __init__ (self, columns, primary_key, tuples=[])

    def columns (self)

    def primary_key (self)

    def create_tuple (self,tup)

    def read_tuple (self,pkey)

    def delete_tuple (self,pkey)

We focus only on the basic CRUD operations here.

Question 3

Complete the implementation of class RelationIndexed which uses a File as storage for tuples. The tuples in the file should be sorted by increasing order of primary key values, like in the RelationSorted case in Question 2. The difference here is that the first block of the file should contain the index. For simplicity, take the index to be a single tuple, of arbitrary length (yeah, yeah, we're simplifying a lot of stuff here). That tuple has, in tuple position i, the value of the primary key of the first tuple in block i+1 of the file (since the first block of the file contains the index, the content proper starts at block 1).

You only have to complete the implementation of create_tuple and read_tuple for this question. Don't implement binary search for read_tuple and create_tuple -- instead, use the index to find the block where the tuple should be found or inserted. The actual insertion for create_tuple can reuse the code for RelationSorted, obviously. Note that create_tuple will force you to update the index.
