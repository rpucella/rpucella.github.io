Course Project

The project for this course should exercise some or all the topics we have covered, from parsing surface syntax to manipulating internal representation to execution via interpretation or via compilation to a lower-level representation.

Ideally, a project should reflect a topic that you are interested in either because it is spikes your curiosity (I really like how fizzbin is done in my favorite language GizmoScript, I wonder how it's implemented) or because it relates to another project you might be involved in from another course (I really need a way to remotely program my spanking new design for an internet-enabled coffee-machine/waffle-maker monstrosity.)

The description of the project that you settle on is due March 28 (see timeline below). In the meantime, you should think about what you might like to do, and send me an email about your thoughts so we can converge on something that is doable in a month but remains interesting. I can also suggest things to read that bear on the topic you are interested in. 

If you do not have any great ideas, I will list some suggestions below in the coming days, and add to it as new ideas come to me.

Projects should be done in teams of 2 or 3. (It is difficult to get something reasonable individually in the time allotted.)

Projects need not be implemented in SML. (Though doing so allows you to reuse the code we've been working with in class and in homeworks.)


Timeline

March 28, 2014 - First deliverable: A short description of the project (at most two pages) including general approach to be taken, preliminary design, sample results or motivating examples.

April 11, 2014 - Second deliverable: A viability prototype for your project. This can be either a proof-of-concept end-to-end artifact, or the implementation of partial functionality for your final deliverable. 

April 25, 2014 - Final deliverable: Your final implementation

Last week of lectures - Class presentations of the projects by the various teams


Suggestions

If any of the projects below interest you, I can supply appropriate reading material to guide you.
  • Create a tool for automatically producing a simpler lexer and parser given a description of a grammar.
  • Investigate a static type system for a programming language with interesting features and implement it; investigate type inference. (Interesting features include polymorphic data structures, higher-order functions, subtyping.)
  • Implement an array-processing language such as APL (or J).
  • Implement a matrix processing language such as Matlab.
  • Implement an existing subset of a specific programming language the implementation of which you are curious about.
  • Push ahead with the production of assembly language in the compilation process for one of the languages we studied.
  • Implement exception mechanisms for the languages we studied.
  • Implement a logic-programming language.
  • Find ways to improve the running-time of the stack machine code obtained from a functional language.
  • Investigate probabilistic programming languages.
  • Implement a language for interactive animations (cf: FRAN)
