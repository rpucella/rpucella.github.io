Databases (Spring 2025)

This is an introduction to database systems. Topics covered include: the relational model, SQL-based querying, transactions, data structures, as well non-relational database models. The approach will be hands-on: while we will practice on existing database systems, we will also write code to implement simplified versions of the various database models and their query systems.

  • Feb 24: SQL (Structured Query Language)

    Reading: Textbook, Chapter 7.

    Slides from the lecture.

    Wikipedia page on SQL.

    A draft of the 1992 standard, if you like reading those sort of documents.

    There are tons of tutorials on SQL online, usually tied to a particular database implementation, since you can't use SQL "in the abstract". I've used Postgres's SQL tutorial in the past to introduce SQL to colleagues (start with Section 2). An interactive tutorial that I greatly enjoy is SQLBolt.

    Two SQL features we did not cover in class but you may run into are views (which are a form of live query that looks like a table in the database) and stored procedures (which are a form of functions that can be written directly into the database). Views tend to look the same across relational databases, while stored procedures are much more vendor dependent.

  • Mar 3: Programming APIs and ORMs

    Reading: Textbook, Chapter 6.2 (normalization) and Chapter 15 (Programming APIs).

    A reasonable set of slides on normalizations and functional dependencies. It goes into a more details than I covered, including decomposition.

    The sample code I demo-ed in class, a simple flask-based web application server that navigates the simple SQLite books database we have been using to study SQL. Unzip the file, go into the database-apis/ folder that gets created, and look at the README.md file to run the code. It requires Python 3.

    JDBC is the Java API for connecting to databases. Most databases have JDBC drivers that support that API. It has become a de facto standard, and many other languages use that same API to provide database connectivity.

    ORMs (Object-Relational Mappings) are a more abstract way to provide an interface to a database through a programming language: ORMs reify database tables as data structures in the language, so that modifying the structures will automatically update the underlying tables, and changes to the underlying tables will be visible through the structures. SQLAlchemy is a popular Python ORM, while GORM is one you can use in Go.

    LINQ is one approach to integrate SQL-style querying within a programming language (here, C#).
